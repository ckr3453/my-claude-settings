# Design Principles for Prototypes

목업 생성 시 AI가 참조할 기본 디자인 원칙.

## 정보 밀도 (Information Density)
- 한 화면에 너무 많은 정보 X
- 가장 중요한 것 1개, 중요한 것 3-5개, 나머지는 숨김/드롭다운
- 빈 여백은 가치가 있음 (불안해서 채우지 말 것)

## 시각 위계 (Visual Hierarchy)
- 크기 > 색상 > 위치 순으로 주목도 결정
- 제목은 본문보다 1.5-2배 크기
- 주요 액션 버튼은 한 화면에 1개만 강조색

## 타이포그래피
- 시스템 폰트 사용 (`font-sans`)
- 본문 16px (`text-base`)
- 줄 높이 1.5 (`leading-relaxed`)
- 한 줄 당 45-75자 (`max-w-prose`)

## 색상
- Tailwind 기본 팔레트 활용
- Primary는 파란색 계열 기본 (`bg-blue-600`)
- 경고는 `text-red-600`, 성공은 `text-green-600`
- 배경은 무조건 흰색 또는 `bg-gray-50`

## 상태 케이스 (필수 포함)
- 빈 상태 (empty state): 데이터 없을 때 무엇을 보여주는가
- 로딩 상태: 스켈레톤 또는 스피너
- 에러 상태: 명확한 메시지 + 복구 액션

## 한국어 타이포그래피 주의
- 한국어는 영문보다 높이가 큼 — `leading-7` 이상 권장
- 긴 단어 줄바꿈 고려: `break-keep` 활용

## 반응형
- 모바일 퍼스트 (`sm:`, `md:`, `lg:` 순)
- 주요 브레이크포인트: 640px (sm), 768px (md), 1024px (lg)
- 모바일에선 사이드바 대신 하단 서랍 또는 햄버거 메뉴

## 접근성 (최소)
- 시맨틱 HTML (`<nav>`, `<main>`, `<article>`)
- 이미지엔 `alt` 속성
- 색만으로 정보 전달 X (아이콘/텍스트 병행)
