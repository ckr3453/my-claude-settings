# 소크라틱 인터뷰 프레임워크

새 프로젝트 또는 새 로드맵 시작 시, 태스크 생성 전에 요구사항을 구체화하는 질문 프레임워크.

---

## 5개 질문 카테고리

### ① 본질 (Essence) — 항상 첫 번째

- "이것의 본질은 뭔가요? 표면적 요청과 실제 목표가 같은가요?"
- "이걸 왜 지금 하려는 건가요? 어떤 문제가 촉발했나요?"
- "이걸 안 하면 어떻게 되나요?"

### ② 근본 원인 (Root Cause) — 버그/개선 요청 시

- "이게 근본 원인인가요, 증상인가요?"
- "이 문제가 처음 발생한 지점은?"
- "같은 문제가 다른 형태로도 나타나나요?"

### ③ 숨겨진 가정 (Assumptions) — 접근 방식이 이미 정해져 있을 때

- "당연하다고 생각하는 전제가 있나요?"
- "다른 방법을 고려해봤나요? 왜 이걸 선택했나요?"
- "사용자/클라이언트가 이미 동의한 부분은?"

### ④ 기존 맥락 (Existing Context) — 브라운필드 프로젝트

- "현재 코드베이스에서 관련된 부분은?"
- "비슷한 기능이 이미 존재하나요?"
- "건드리면 안 되는 것은?"

### ⑤ 완료 조건 (Completion) — 항상 마지막

- "어떻게 되면 '완료'인가요? 구체적으로."
- "누가, 어떻게 검증하나요?"
- "성능/품질 기준이 있나요?"

---

## 적응적 규칙

- **본질로 시작. 항상.**
- 답변이 이미 구체적이면 완료 조건으로 건너뜀
- 기존 시스템 언급 시 → 기존 맥락 먼저
- 라운드당 질문 1~2개. 한꺼번에 덤프 금지
- **최소 3라운드 필수. 상한 없음.**
- 사용자가 "충분해" / "진행해" / "됐어"라고 하면 종료
- AI도 충분히 명확하다고 판단하면 종료 제안 가능 ("충분히 명확한 것 같은데, 스펙을 정리할까요?")
- 라운드 사이에 Glob/Read로 코드베이스 탐색하여 질문 근거 확보
