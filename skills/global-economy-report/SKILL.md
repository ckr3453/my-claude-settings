# Global Economy Report Skill

세계 경제 이슈를 수집, 분석하여 마크다운 보고서 생성 후 이메일 전송.

---

## ⛔ 날짜 필터링 (최우선 규칙)

**보고서의 모든 항목에 예외 없이 적용. 위반 시 보고서 품질이 심각하게 저하된다.**

### 허용 범위: 오늘 + 어제 (달력 기준, 2일치)
- ✅ URL/제목에 **오늘 날짜** 또는 **어제 날짜**가 있는 기사
- ✅ 상대 시간 "N시간 전", "1일 전" → 포함
- ❌ **"2일 전" 이상 → 무조건 제외**
- ❌ 날짜 확인 불가 → 제외 (의심되면 버려라)
- ❌ 정보가 부족해도 오래된 뉴스로 채우지 말것. 빈 섹션이 오래된 뉴스보다 낫다.

### 제외 대상
- URL에 오늘/어제가 아닌 날짜 (지난달, 작년 등)
- "YYYY outlook", "YYYY 전망" 등 전망/아웃룩 기사
- "최근", "이번 주", "올해" 같은 모호한 시간 표현만 있는 기사
- "2일 전", "3일 전", "지난주" 등 명확히 2일 이상 지난 기사

### 검색 키워드 규칙
- WebSearch 쿼리에 **어제 날짜**를 포함 (검색엔진 인덱싱 지연 때문)
- 형식: `{검색어} {Month DD, YYYY}` (어제 날짜)
- ❌ 금지: "today", "this week", "recent", "latest", 연도만 단독 사용

## 보고서 섹션 구조

보고서는 **해외**와 **국내**를 명확히 구분하여 작성한다.

### 🌍 해외
1. **주요 경제 뉴스** - 글로벌 핵심 경제 이슈
2. **증시** - S&P500, 나스닥, 다우, 유럽/아시아 주요 지수
3. **중앙은행 정책** - Fed, ECB, BOJ 등 통화정책
4. **산업별 동향** - 주요 산업 섹터 뉴스
5. **무역/통상** - 관세, FTA, 국제 통상 이슈
6. **투자 동향** - M&A, IPO, VC 투자 등

### 🇰🇷 국내
1. **주요 경제 뉴스** - 국내 핵심 경제 이슈
2. **증시/환율** - 코스피, 코스닥, 원/달러 환율
3. **한국은행 정책** - 금리, 통화정책
4. **산업별 동향** - 국내 주요 산업 뉴스
5. **무역/통상** - 수출입, FTA, 통상 이슈
6. **투자 동향** - 국내 M&A, IPO, VC 투자 등

### 공통
- **추천 읽을거리** - 국내외 심층 기사/분석 (중복 금지)

## 실행 시작 전 필수 확인

### Step 0: 날짜 확인 (모든 작업 전 반드시 수행)

**Bash로 오늘/어제 날짜를 계산한다:**
```bash
date "+%Y-%m-%d"              # 오늘 ISO (예: 2026-03-12)
date -v-1d "+%Y-%m-%d"        # 어제 ISO (예: 2026-03-11)
date "+%B %d, %Y"             # 오늘 Full (예: March 12, 2026) - 검색 쿼리용
date -v-1d "+%B %d, %Y"      # 어제 Full (예: March 11, 2026) - 검색 쿼리용
date "+%Y/%m/%d"              # 오늘 URL 패턴 (예: 2026/03/12)
date -v-1d "+%Y/%m/%d"       # 어제 URL 패턴 (예: 2026/03/11)
```

**이 값들을 변수처럼 기억하고 이후 모든 검색/필터링에 사용한다.**
- 검색 쿼리: 어제 Full 형식 사용
- URL 검증: 오늘/어제 URL 패턴으로 매칭
- 상대 시간: "N시간 전" ✅, "1일 전" ✅, "2일 전" ❌

---

## ⚠️ 에러 처리 원칙

### 순차 실행 필수
- ❌ **병렬 호출 금지**: WebFetch/WebSearch를 동시에 여러 개 호출하면 하나 실패 시 전체 실패
- ✅ **순차 실행**: 한 번에 하나씩 호출하여 에러 격리
- ✅ **실패 시 계속**: 한 소스 실패해도 다음 소스로 진행

### 요청 간 간격
- WebFetch/WebSearch 호출 사이에 **최소 3초 간격** 유지
- 연속 요청 시 rate limiting 방지 목적

### 재시도 정책
- WebFetch 실패 시 **5초 대기 후 1회 재시도**
- 재시도도 실패 시 → **WebSearch 폴백** (해당 매체명 + 날짜로 검색)
- WebSearch 폴백 예시: `사이트:edaily.co.kr 경제 뉴스 {오늘 날짜}` 또는 `"이데일리" 경제 {어제 날짜}`
- WebSearch 폴백도 실패 시 → 해당 소스 건너뛰고 다음 진행

### 에러 대응
- 특정 소스 접근 실패 → 재시도 → 폴백 → 건너뛰고 다음 소스 진행
- 검색 결과 없음 → 해당 섹션 생략 (정상)
- 전체 실패 → "오늘은 데이터를 수집할 수 없습니다" 보고서 생성

## 실행 프로세스

### Step 1-A: 국내 경제 뉴스 크롤링 (WebFetch - 순차 실행)

**중요: 반드시 순차적으로 하나씩 호출할 것 (병렬 호출 금지)**

**⚠️ WebFetch 프롬프트 핵심 규칙:**
- WebFetch 내부의 소형 모델은 **현재 날짜를 모른다**
- 따라서 프롬프트에 **반드시 오늘/어제의 실제 날짜를 명시**해야 한다
- `{오늘}`, `{어제}`를 Step 0에서 계산한 실제 날짜로 치환할 것

**공통 프롬프트 템플릿:**
```
"오늘은 {오늘 YYYY년 MM월 DD일}이다. 이 페이지에서 경제, 금융, 증시, 환율, 무역, 산업 관련 주요 뉴스를 찾아서 제목, 날짜/시간, URL을 추출해줘. 반드시 {오늘 MM월 DD일} 또는 {어제 MM월 DD일} 뉴스만 포함하고, 그 이전 날짜의 뉴스는 모두 제외해줘. 날짜를 알 수 없는 뉴스도 제외해줘."
```

**각 소스별 실행 흐름 (공통):**
1. WebFetch 시도
2. 실패 시 → 5초 대기 → WebFetch 재시도 (1회)
3. 재시도 실패 시 → WebSearch 폴백 (`"매체명" 경제 {어제 날짜}`)
4. 폴백도 실패 시 → 스킵하고 다음 소스 진행
5. **다음 소스로 넘어가기 전 최소 3초 대기**

### Step 1-B: 해외 경제 뉴스 검색 (WebSearch - 순차 실행)

**중요: 반드시 순차적으로 하나씩 호출할 것 (병렬 호출 금지)**

**검색 순서 (카테고리별로 1개씩 순차 실행):**

**검색 쿼리에 오늘 또는 어제 날짜 사용 (검색은 넓게 → 결과는 48시간 필터링)**

1. **주요 경제 뉴스** (1개씩 순차):
   - WebSearch: `Reuters global economy news [어제 날짜]`
   - WebSearch: `Bloomberg economy news [어제 날짜]`
   - WebSearch: `CNBC economy market news [어제 날짜]`
   - WebSearch: `WSJ economy news [어제 날짜]`
   - WebSearch: `MarketWatch market news [어제 날짜]`

2. **증시** (1개씩 순차):
   - WebSearch: `S&P 500 Nasdaq stock market [어제 날짜]`
   - WebSearch: `KOSPI stock market [어제 날짜]`

3. **중앙은행/무역/투자** (필요시 추가 검색):
   - WebSearch: `Federal Reserve policy [어제 날짜]`
   - WebSearch: `US China trade [어제 날짜]`
   - WebSearch: `major M&A IPO [어제 날짜]`

**각 검색마다:**
- 성공 → 결과 검토 후 48시간 필터링 적용
- 실패 → 에러 로그 후 다음 검색 진행
- 결과 없음 → 정상, 다음 검색 진행

**예시: 오늘 "{Month DD, YYYY}", 어제 "{Month DD, YYYY}"**
→ Reuters global economy news {어제 Month DD, YYYY}
→ Bloomberg economy news {어제 Month DD, YYYY}

**왜 어제 날짜로 검색하는가?**
검색 엔진은 당일 기사를 인덱싱하는 데 수시간~1일 지연이 있다.
어제 날짜로 검색하면 실제로는 오늘까지의 최신 기사가 더 많이 잡힌다.

### Step 1-C: 날짜 필터링 (오늘/어제 기준)

수집한 모든 항목에 대해:
1. URL에 Step 0의 오늘/어제 날짜 패턴이 있는가? → ✅
2. "N시간 전", "1일 전" 표시가 있는가? → ✅
3. "2일 전" 이상, 날짜 불명확, 오래된 날짜 → ❌ 제외

**의심스러우면 제외. 빈 섹션이 오래된 뉴스보다 낫다.**

3. **중복 제거**: 같은 뉴스는 가장 신뢰도 높은 1개만 채택. 동일 URL 2번 등장 금지. 추천 읽을거리에는 다른 섹션 항목 절대 금지
4. **섹션 생략**: 48시간 이내 뉴스가 없는 섹션은 생략 (빈 섹션 > 오래된 뉴스)
5. **내용 없으면 솔직하게**: "오늘은 특별한 소식이 없습니다" 표기
