# PRD Question Bank

prd-writer 인터뷰에서 AI가 선택해 사용할 질문 풀.
카테고리별로 정리되어 있으며, 각 라운드별 2-4개씩 선택해 사용한다.

---

## 1. 문제 정의 (Socratic)

- "이걸 왜 지금 만들려 하시나요?"
- "이걸 안 만들면 어떻게 되나요? 구체적인 결과는?"
- "이 문제가 새로 생긴 건가요, 오래된 건가요? 왜 그동안 해결되지 않았나요?"
- "이미 존재하는 해결책은 없나요? 있다면 왜 부족한가요?"
- "이 문제를 가장 크게 겪는 사람은 누구인가요?"

## 2. 본질 파악 (Ontological)

- "이게 본질적으로 무엇인가요? 한 문장으로 정의한다면?"
- "증상을 해결하는 건가요, 원인을 해결하는 건가요?"
- "이것 없이도 같은 결과를 낼 수는 없나요?"
- "이걸 만든다는 건 '무엇을 만든다'는 것이 아니라 '무엇을 바꾼다'는 의미인가요?"
- "다른 형태로는 해결할 수 없나요? (예: 기능 추가 대신 기존 기능 제거)"

## 3. 유저와 맥락

- "이걸 쓸 사람 한 명만 구체적으로 떠올려 주세요. 이름, 직무, 상황까지."
- "그 사람이 지금 이 순간 무엇을 하고 있나요?"
- "이 기능을 만나는 순간 그 사람의 감정은 어떤가요? 바쁨, 막힘, 궁금함?"
- "그 사람이 이 기능을 쓰기 직전과 직후, 무엇이 달라지나요?"
- "이 기능을 절대 쓰지 않을 사람은 누구인가요? 왜인가요?"

## 4. 범위와 MVP

- "MVP에서 뺄 수 있는 건 무엇인가요?"
- "만약 개발 시간이 반으로 줄어든다면 뭘 포기하시겠어요?"
- "이게 출시된 첫날, 유저가 반드시 할 수 있어야 하는 한 가지는?"
- "버전 2에서 추가해도 되는 건 무엇인가요?"
- "이 제품이 **아닌** 것은 무엇인가요?"

## 5. 가정 검증 (Elicit)

- "우리가 당연하게 여기고 있는 가정 3개를 꼽아 주세요."
- "각 가정이 틀렸다면 어떻게 되나요?"
- "가장 위험한 가정은 무엇이고, 그걸 가장 싸게 검증할 방법은?"
- "과거에 비슷한 걸 시도한 사례가 있나요? 결과는?"
- "이 가정이 틀려도 상관없는 영역은 어디인가요?"

## 6. 성공 지표

- "이게 성공했다는 걸 어떻게 알 수 있나요?"
- "수치로 표현한다면? 언제까지 어떤 수치를 달성하면 성공인가요?"
- "실패의 정의는 무엇인가요? 어떤 상태가 되면 철수해야 하나요?"
- "성공과 실패를 가르는 단 하나의 지표를 고른다면?"

## 7. 화면 & 플로우 (UI가 있는 피처만)

- "유저가 처음 이 기능을 만나는 화면은 어떤 모습이어야 하나요?"
- "가장 중요한 동선 한 줄로 기술해 주세요. (A 화면 → B 화면 → ...)"
- "빈 상태(데이터 없음), 로딩, 에러 상태는 어떻게 보여야 하나요?"
- "모바일과 데스크톱 중 어느 쪽이 1순위인가요?"

---

## AI 사용 지침

1. 각 라운드별 2-4개만 선택. 한 번에 몰아 묻지 말 것.
2. 사용자 답변에 맞춰 카테고리 넘나들기 허용 (예: 유저 질문 중 가정이 드러나면 5번으로 점프)
3. 이미 답변이 충분한 카테고리는 재질문 금지
4. 카테고리 7번은 UI 있는 피처만. 백엔드/인프라 피처는 스킵
5. 질문 자체를 그대로 읽지 말고, 사용자 맥락에 맞게 자연스럽게 변형
