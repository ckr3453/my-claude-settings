# Mode B: 인터뷰 질문 흐름

## 개요

사용자와의 대화를 통해 블로그 글감을 발굴한다. git 이력이 없거나, 코드 외 경험을 블로그로 쓰고 싶을 때 사용.

## 인터뷰 단계

### Phase 1: 오프닝 (1개 질문)

`AskUserQuestion`으로 시작:

```
"최근에 개발하면서 기억에 남는 경험이 있나요?
 삽질, 새로 배운 것, 기술 선택 고민 등 뭐든 좋습니다."
```

**옵션 예시:**
- "삽질/버그 해결 경험이 있다"
- "기술 선택을 고민했다"
- "새로 알게 된 것이 있다"
- "프로젝트/기간을 돌아보고 싶다"

### Phase 2: 유형 감지

오프닝 응답에서 포스트 유형을 추정:

| 응답 키워드 | 추정 유형 |
|------------|----------|
| 에러, 버그, 안 됨, 삽질, 해결 | 삽질기 |
| 선택, 비교, 고민, A vs B | 의사결정기 |
| 알게 됐다, 발견, 처음 알았다 | TIL |
| 돌아보니, 회고, 마무리, 끝났다 | 회고 |

### Phase 3: 후속 질문 (3~5개)

유형에 따라 다른 질문 세트를 사용. `AskUserQuestion`으로 1개씩 순차 진행.

#### 삽질기 질문 세트

1. "어떤 에러/문제가 발생했나요? 에러 메시지나 증상을 기억나는 대로 알려주세요."
2. "처음에 어떻게 해결하려고 했나요? (첫 번째 시도)"
3. "최종적으로 어떻게 해결했나요? 원인은 뭐였나요?"
4. (선택) "해결하는 데 얼마나 걸렸나요?"
5. (선택) "이 경험에서 배운 교훈이 있나요?"

#### 의사결정기 질문 세트

1. "어떤 선택을 해야 하는 상황이었나요?"
2. "어떤 옵션들을 고려했나요?"
3. "최종적으로 뭘 선택했나요? 결정적인 이유는?"
4. (선택) "실제로 써보니 어땠나요?"
5. (선택) "다시 선택한다면 같은 걸 고르겠어요?"

#### TIL 질문 세트

1. "뭘 하다가 이걸 알게 됐나요?"
2. "알게 된 내용을 간단히 설명해주세요."
3. "이전에는 어떻게 하고 있었나요? (비교가 있으면)"
4. (선택) "이걸 알면 어디에 유용한가요?"

#### 회고 질문 세트

1. "어떤 프로젝트/기간에 대한 회고인가요?"
2. "가장 잘했다고 생각하는 점은?"
3. "가장 아쉬운 점은?"
4. "기술적으로 배운 것이 있나요?"
5. (선택) "다음에 다르게 하고 싶은 것은?"

### Phase 4: 주제 후보 생성

인터뷰 결과를 종합하여 **2~3개 주제 후보** 생성.

```
1. [삽질기] "OOM 에러와 싸운 3시간"
   - 핵심: Redis 메모리 관리 미스로 인한 OOM
   - 포인트: maxmemory-policy 설정 누락

2. [TIL] "Redis maxmemory-policy의 6가지 옵션"
   - 핵심: 각 옵션의 차이와 적합한 상황
   - 포인트: volatile-lru vs allkeys-lru
```

같은 소재라도 다른 유형으로 제시하면 선택지가 넓어진다.

## 인터뷰 원칙

- **한 번에 1개 질문**: 여러 질문을 한꺼번에 던지지 않는다
- **응답 존중**: 사용자 답변을 그대로 소재로 활용 (재해석하지 않음)
- **강요하지 않음**: "(선택)" 질문은 "패스해도 괜찮습니다" 옵션 제공
- **자연스러운 마무리**: 충분한 소재가 모이면 질문을 멈춤 (최소 3개, 최대 5개)
- **전환 가능**: 인터뷰 중 git 이력 확인이 필요하면 Mode A 부분 활용 가능
