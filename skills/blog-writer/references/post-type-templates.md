# 포스트 유형별 템플릿

4가지 유형의 기술 블로그 포스트 골격.

---

## 1. 삽질기 (Struggle)

**핵심**: 문제 해결 과정을 시간순으로 생생하게 전달

```markdown
## 발단

{무엇을 하고 있었는지 배경 1~2문장}
{어떤 문제가 터졌는지 1문장}

## 첫 번째 시도

{구글링/직감으로 시도한 내용}

​```{lang}
{시도한 코드 또는 명령어}
​```

{결과 — 보통 실패}

## 두 번째 시도

{다른 접근 방식}
{왜 이 방법을 시도했는지}

{결과}

## 해결

{최종 해결 방법}

​```{lang}
{해결 코드}
​```

{왜 이게 답이었는지 원인 분석}

## 배운 것

{1~3개 핵심 교훈}
{앞으로 주의할 점}
```

**사용자 확인 포인트**: 시도 과정 중 빠진 에피소드 없는지

---

## 2. 의사결정기 (Decision)

**핵심**: 왜 이 기술/방법을 선택했는지 논리적 흐름

```markdown
## 상황

{프로젝트/팀 배경}
{해결해야 할 문제 또는 요구사항}

## 선택지

{고려한 옵션들 나열}

| | {옵션 A} | {옵션 B} | {옵션 C} |
|---|---------|---------|---------|
| {기준 1} | ... | ... | ... |
| {기준 2} | ... | ... | ... |
| {기준 3} | ... | ... | ... |

## 고민 과정

{각 옵션의 장단점을 팀/프로젝트 맥락에서 분석}
{결정적으로 기울게 만든 요인}

## 결정

{최종 선택과 이유}

## 결과

{실제로 적용해본 소감}
{예상과 달랐던 점}
{다시 선택한다면?}
```

**사용자 확인 포인트**: 비교 기준과 고민 포인트가 실제와 맞는지

---

## 3. TIL (Today I Learned)

**핵심**: 짧고 밀도 있게, 하나의 발견

```markdown
## 계기

{무엇을 하다가 이걸 알게 됐는지 1~2문장}

## 핵심

{알게 된 내용 설명}

​```{lang}
{핵심을 보여주는 코드 예시}
​```

{코드 설명 1~2문장}

## 왜 중요한가

{이 지식이 어디에 쓸모 있는지}
{모르면 생길 수 있는 문제}
```

**사용자 확인 포인트**: 핵심 내용이 정확한지

---

## 4. 회고 (Retrospective)

**핵심**: 일정 기간/프로젝트를 돌아보며 성장 포인트 정리

```markdown
## 배경

{회고 대상: 프로젝트명, 기간, 목표}
{시작할 때의 상황}

## 잘한 것

{구체적 사례 2~3개}
{왜 잘했다고 생각하는지}

## 아쉬운 것

{구체적 사례 2~3개}
{원인 분석}

## 배운 것

{기술적 교훈}
{프로세스/협업 교훈}

## 다음에는

{개선할 구체적 행동 1~3개}
```

**사용자 확인 포인트**: 잘한 것/아쉬운 것이 빠짐없이 반영됐는지

---

## 유형 자동 감지 힌트

| 신호 | 유형 |
|------|------|
| 에러 해결, 디버깅, 원인 분석 | 삽질기 |
| 기술 선택, A vs B, 비교 | 의사결정기 |
| 새로 알게 된 것, 짧은 발견 | TIL |
| 기간 회고, 프로젝트 마무리 | 회고 |
| git diff에 다양한 시도 흔적 | 삽질기 |
| 설정/의존성 변경 위주 | 의사결정기 |
| 단일 파일 소규모 변경 | TIL |

유형이 모호하면 사용자에게 2개 후보를 제시하고 선택받는다.
